# 想法纪要(IDEAS)

> 作者理解选题过程中的类比、顿悟与自我纠错记录。按日期追加,每条注明"对在哪 / 边界在哪"。
> 用途:答辩时的直觉弹药库 + 防止同一个坑掉两次。

## 2026-07-21 · 概念地基期(alpha 因子从零到闭环)

### 类比与心智模型

- **王者荣耀高端局**:基本功人人拉满(≈公共收益 β/行业/风格,人人可得),赢靠找对手没防住的破绽(≈alpha,未被定价的残余规律)。加强版:破绽会被打补丁——alpha 会拥挤、IC 会衰减,所以"生产假说的速度"本身是竞争力,这正是 agent 选题的立足点。
- **赌场 / 硬币偏置**:IC=0.05 ≈ 51.5:48.5 的硬币。单次抛掷看不到偏置,聚合(每天几千只股票 × 几百个交易日)才显形。alpha 是硬币的偏置,不是某一次抛掷的结果;单只股票的 alpha 测不准、也从不去测。
- **制造可比性三兄弟**:折现(不同时点的钱 → 同一时点可比)、zscore(不同量纲 → 同尺度可比)、中性化(不同行业 → 同类内可比)。目的同构、过程完全不同。
- **傅里叶变换 ↔ 中性化**:同为"投影—分解—取正交残差"。中性化 = 把因子信号里的行业频率、市值频率滤掉的带阻滤波器。边界:行业/市值这组基是每天估计的、不正交、不能完美重构。
- **TDD ↔ 假说先行**:先写可证伪断言再验证,精神一致。**关键反差**:TDD 里改代码到测试通过是美德;因子研究里改因子到回测通过是 p-hacking。回测数据只有一份,循环每多跑一轮可信度掉一分。
- **吃肉喝汤(已纠正)**:β 才是汤(量大廉价,指数基金批发),alpha 才是肉(稀缺,2+20 的定价)。公共部分不是"别人不要的",alpha 也不是边角料——所有人都在抢肉,我们赌的是挖掘工具的效率。

### 关键自我纠错(掉过的坑)

- "alpha 因子 = 找 AR 趋势明显的序列" ❌ → 反例:E/P 价值因子与自身收益序列毫无关系。特征沿时间算,预测力沿横截面验;动量只是长得像 AR 的特例。
- "先有因子再让 AI 解释" ❌ → 顺序必须反过来:解释是因子的出生证,不是事后补办。事后解释不可证伪(LLM 能给任何公式编故事),事前假说可被回测证伪。
- "agent 优势在非结构化数据处理" ❌ → 本项目三张表全是结构化数据。LLM 的真实优势:带经济先验的假说生成、自然语言↔公式双向翻译、语义查重。LLM 管语义空间,代码管数值空间。
- "我们不考虑基本面" ❌ → 一半基线因子(价值 E/P、质量 ROE)就是基本面因子。正确表述:系统化使用基本面数据,不做个股层面的主观基本面判断。
- "营收来源于成交价" ❌ → 两个市场:营收来自商品市场(卖酒),成交价来自股票市场(炒股)。边界判据:这个数字是否由"股票被交易"产生。
- "alpha 是组合比行业优越的原因" ❌ → alpha 是结果(扣完公共账目剩下的钱),因子才是原因(信号),解释是机制。因果链:因子 → 选股 → 收益 → 记账分解 → alpha。
- "用测试集调参" ❌ → 训练段随便看随便调;测试段封存,最后拆封跑一次,结果直接进报告。
- "测试结果用来判断要不要继续" ❌(更严!)→ 测试集连"继续/停止"的决策都不许参与,否则是选择偏差(跑 N 个因子专挑 OOS 过线的 = 挑运气)。它只贡献一件事:给最终结论盖章。OOS 挂了 = 因子死了,如实报告;要决策信号就切三段(训练/验证/测试)。

### 答辩可用的一句话

- 时间序列分析预测"一只股票的明天",因子投资预测"今天谁比谁强";所以评价指标是逐日横截面相关,不是时间序列拟合优度。
- 不是相信世界线性,而是 R² 千分之几的信噪比下,线性是唯一学不了噪声的模型;且用排名后"线性"已放松为"单调"。
- LLM 负责语义空间(假说/翻译/定性反馈),代码负责数值空间(计算/评价/裁决);可信度来自假说的生杀大权在不含 LLM 的回测手里。
- 回测的第一美德不是收益高,是诚实还原每个历史时刻的信息集(pit_merge 按 ann_date 对齐)。
- 反馈环:限制信息的时间范围(只看训练段),不限制详细程度(段内明细全开)——效率来自细节,可信度来自封存。
- 漏斗即结论:"生成 N 个假说 → 训练段过 m 个 → 样本外站住 k 个",负结果也是结果。
- **信息到测试集必须是单向的**:训练的知识可以流向测试(带着所学去赴考),测试的结果绝不能流回任何决策——哪怕只是"瞟一眼、心里有个数"也算泄漏,因为泄漏经过大脑一次就完成了,不需要走正式流程。封存不是仪式,是物理隔离。(本日最终顿悟,由"测试效果若被参考,就成了定性指标"推出)

### 流程优化清单(研究纪律,落地到闭环实现里)

1. **反馈环设迭代上限**:每个假说最多改 2 轮,改不好就弃,换新假说——不许死磕到通过。
2. **选平台不选尖峰**:调参看参数邻域的平坦度(20/40/60 天都在 0.03~0.04 才可信),孤立尖峰(37 天最好)是过拟合自白。
3. **经济逻辑当否决票**:每次改动必须讲得出理由("拉长窗口为降换手"),讲不出就不改。
4. **反馈粒度约定**:LLM 只看训练段,但训练段明细全开(IC/ICIR/分年度/分组单调性/换手/成本后收益)——限制时间范围,不限制详细程度。
5. **OOS 一次性拆封**:测试段结果只给最终报告盖章,不参与任何决策(包括"继续/停止");挂了就如实报告因子死亡。
6. **需要决策信号就切三段**:训练(随便调)/ 验证(负责继续与否)/ 测试(只盖章)。本项目两段够用,答辩被追问时给出三段方案。
7. **漏斗即核心图表**:"生成 N 假说 → 训练段过 m → 样本外站住 k",这张图是答辩的主结论,负结果也是结果。

## 2026-07-21 · 展示面壁垒的三次崩塌(设计盘问记录)

设计"知识图谱 + 成果展示"时,壁垒方案被自己推翻了三次,每次都是**发现前提错了**而不是方案不好。记下来,因为这条推理链本身就是答辩可用的素材。

### 崩塌一:留微信解锁 = 伪命题

**用户顿悟:**"能让他们看到我的网站,应该一定有他们的联系方式了。"

→ 靠"填联系方式"换访问权的门,在**受众本就是熟人**的场景下收集不到任何新信息。它不是获客,顶多是访客簿。
**推广:任何"以信息换访问"的设计,先问受众是不是陌生人。不是,则该设计的收益项为零。**

### 崩塌二:项目会终结 → 权限体系的经济基础消失

**用户一句话掀桌:**"答辩结束我项目就结束了啊。"

→ 四层权限(公开/软门/口令/永不出)保护的是**未来的持续价值**。项目终结 = 没有未来价值 = 保护对象不存在。
同理,版本浏览器需要"有很多版本"才成立;还剩 3 天、总共 2 个版本,为一条两行的时间线建 tag+manifest+milestone 全套基建,是拿基础设施的成本买不存在的规模。
**推广:一切基础设施的价值 ∝ 它未来被使用的次数。先估次数,再决定建不建。**

### 崩塌三:红线安全 ≠ 防抄安全

论文全篇合成数据 + mock 因子,**对"别漏真 alpha"这条红线是安全的**——之前据此判定可以公开全文。
但站点的首要动机是**给同学看**,而壁垒的真实含义是"别让同学在他们答辩前抄走方法"。**论文正文就是完整方法论**——对红线安全,对防抄是送人头。
**两套判断被混为一谈了。同一份材料,换一个威胁模型,结论完全相反。**

### 最终收敛:最强的壁垒是"不上传"

> `build.py` 只渲染写进内容文件的东西。**没写进去的,服务器上根本不存在。**

任何 token 体系都比不过"它不在那儿"。三天期限内,curation(策展)严格优于 access control(访问控制):成本为零、无运行时失败、无绕过面。
配套两条:内容开关**默认 false**(fail-closed);**禁止把未公开内容渲染成模糊占位块**——那等于宣告"这里有东西",反而招惹。

### 附带两个正向发现

- **知识图谱不是装饰,是论点的证据。** 论文写着"可解释性可一路回溯到具体字段与有限算子",而图正好把这句话画出来。它一鱼三吃:站点交互图 / PPT 一页 / 论文配图。**图证明的是"方法可审计",不是"因子是什么"**——所以因子层只放类别不放表达式,炫耀价值高、被抄价值低。
- **工单挂 milestone 徽章,把意见箱变成有回响的参与感。** 同学几天后回来看到"我提的建议被做进 v0.3 了"——这比展示成果本身更能带回访。而 GitHub API 响应里本来就带 milestone 字段,**改一行**。

## 2026-07-22 · 观测方式制造的假象(方法图调试记录)

调方法图那一晚,我连着两次判定"这里有 bug",两次都错——**错的不是代码,是我的观测方式**。记下来,因为这类错误在做交互式/滚动驱动的东西时会反复出现,而且它和研究里的一类错误同构。

### 假象一:把系统锁在初态,然后说它坏了

方法图是**滚动驱动展开**的:滚过一段"跑道"时,画布从版心宽 720px 撑到 930px(`--gx` 从 0 到 1)。
我为了截图方便,把前面的区块全 `display:none`、再 `scrollTo(0,0)`——等于把系统**钉死在折叠初态**,然后看到画布只有 720px、图放不下,判它"全宽失效"。

**测试环境的简化把被测行为本身消掉了。** 隐藏前面的区块改变了滚动几何,归零滚动条抹掉了驱动信号——我测的已经不是那个系统了。

### 假象二:同步脚本量异步状态

第二次我改用真实滚动:一段脚本里连滚 5 个位置、每滚一次读一次 `--gx`,结果全是 0,又判"滚动展开没生效"。

但展开逻辑走的是 `requestAnimationFrame` —— **rAF 回调在同步脚本执行期间根本没有机会运行**。我在一个从未被推进过的世界里连拍了五张快照,然后说这个世界不会动。
分成两次调用(中间让浏览器喘一口气)再量,`--gx` 立刻是 1,画布 930px,一切正常。

### 和研究里的错误是同一个

这两条本质都是:**用一个破坏了因果链的观测,去否证被观测对象**。
和"用测试集调参"是镜像关系——那边是观测污染了系统,这边是观测阉割了系统。共同点是:**在下结论前,先问"我的测量装置有没有把要测的东西本身改掉/关掉"。**

回测里对应的具体形态:改了一个参数后指标"没变化",先别急着说这个参数没用——**先确认改动真的进入了那次运行**(缓存没失效?panel 没重算?样本外区间没被空掉?)。很多"这个因子没效果"其实是"这次运行根本没用上这个因子"。

### 顺带:一个漂亮的约束求解

图有 5 层放不进 760px 的版心,我原来的解法是让它横向滚动——**等于把问题推给用户**。
Codex 的解法是两头夹:**把"评价指标"整列抽出来做成浮贴卡片**(画布 1148→916)+ **滚动展开**(版心 720→930)。916 < 930,横向滚动彻底消失。
教训:**当一个东西"放不下"时,除了扩大容器,还可以重新决定什么必须在容器里。** 指标层本来就不是依赖链的一环,而是链的终点——它不需要和字段/算子挤在同一张网格里。
