# 研究工作流设计 — 让面板围绕一条正式流程串起来

> 作者:周梓煜　草拟:2026-07-24
> 定位:本文是流程骨架设计,凌驾于 `docs/AGENT_PANEL_SPEC.md`(面板实现)与 `docs/PLATFORM_V2_SPEC.md`(展示站)之上。
> 它回答一个问题:**这些零散功能靠什么串成一个"好用"的软件?** 答案是一条带闸门的正式流程。
> 配套决策见待建的 ADR-0024(工作流状态机)、ADR-0025(录播式工作台)、ADR-0026(拒绝多 Agent 架构)。

---

## 0. 产品命题:这个软件会拒绝你

市面上的回测工具都是"让你随便跑,跑到好看为止"。本工具的价值恰好相反:

> **它在你想再跑一次的时候拦住你,并把"你已经试了几次"显示在屏幕上。**

这和项目立论(系统敢否决自己)同构。流程之所以能把零散功能串起来,是因为**流程里有闸门**;没有闸门的流程只是一串按钮。整个软件区别于任何回测工具的,就是**两处不可逆的门 + 一处不退的预算**,其余都是壳。

## 1. 两个参考来源(方法论 vs 机制)

### 1.1 联合利华洞察流程(方法论内核)

`~/dev/work/miaosi/fang` 的 `insight/` 手记(前联合利华副总裁提出的数据洞察逻辑)提供的不是代码,是**把研究框成对象**的方法论。四条直接映射到本项目:

| 洞察手记原文 | 映射到 Alpha 因子研究 |
|---|---|
| **"研究说明(定义域和值域)"** — 把一次研究框成定义域→值域映射 | **因子即函数**:定义域 = T 日 `ann_date≤T` 可得字段,值域 = 横截面排名。这就是"研究合同"的本体 |
| **"核心观点"写在最前** + "观点 => 结论" | **假说先行**:解释是因子的出生证,不是事后补办(事后解释不可证伪) |
| **"特点总结…类似函数签名"** | `FactorExpr` 元数据(用了哪些字段/算子/经济机制)= 因子的签名 |
| **"每步都能输入,类似非线性对话"** | Canvas / 非线性工作台:任意一步可人工注入,不是死管道 |
| **"人能圈重点,方便用户"** + "先分析现在才能预测未来" | 人工策展 + 先吃透样本内结构再封样本外 |

### 1.2 fang 状态机(机制形态)

`fang` 是多人协作的洞察工作台(基于阶段模板,支持推进/回退/跳过/重开)。**它的方法论我们抄,它的协作机制大多不抄**:

**该抄:**
1. **触发式任务,不是流程段。** fang 的"数据矛盾检查"没有阶段编号,规则是"阶段 7 与 8 都完成后自动创建矛盾检查任务"。→ 我们的鲁棒性检验同理:哪一段都不放,是回测完成后**自动弹出的卡**。
2. **状态机是纯函数,放共享层,带单测。** `PHASE_TEMPLATE` 常量表 + `transitionPhase(current, action)` 纯函数。→ 我们做 `RESEARCH_PHASE_TEMPLATE` + `transition_research_phase()` + pytest。闸门散在 UI 的 if 里等于没有闸门。
3. **统一工作台骨架。** fang 每阶段五区块(说明/任务/数据源/产出/备注),用户只学一次。→ 我们每段五块:这一步在干什么 / 输入 / 正在跑什么 / 产出 / **为什么可信**。第五块是我们独有的。
4. **首屏主线引导(FR-02-04/05):** "先做什么后做什么"三步走,任务/资料/产出**不得并列争抢首屏权重**,次级模块折叠降权。→ 直接借为验收标准,落地"不喧宾夺主"。
5. **审计日志 append-only。** → 尝试预算每次消耗必须 append-only,最终报告"共尝试 N 次"从日志统计,不靠自觉披露,靠没法不披露。
6. **定位那句话,换个名词就是我们的:** 决策支持"给出证据、权衡和不确定性,而不是假装做出权威决策"→ "给出研究证据,而不是假装给出投资建议"。

**不能抄:**
1. fang 阶段全部可回退/跳过/重开(四动作三个往回走)。**我们第 6 段测试拆封一旦执行不可回退** —— 回退它等于允许重跑测试集。我们的状态机必须有 fang 没有的**不可逆迁移**概念。
2. fang 的 apiKey 服务端加密入库。我们 ADR-0023 零后端,key 只能在浏览器 localStorage。
3. fang 阶段 2–8 可并行。我们必须串行,尤其 5→6 绝不并行(等于一边优化一边看测试集)。
4. fang 是多人系统(指派/提醒/逾期/9 权限位),我们是单人研究工具,全不需要。

## 2. 流程主干(七段)

每段五区块统一骨架;下表只列第五块"为什么可信"与迁移性质。

| 段 | 用户看见什么 | 谁在算 | 为什么可信 | 迁移性质 |
|---|---|---|---|---|
| **0 研究线索** | 自由输入的自然语言观察 | LLM 澄清 | — | — |
| **1 研究合同** | 数据域、时间范围、股票池、可证伪断言、**三段切分点** | LLM 起草 + 用户确认 | 切分点冻结,哈希存档 | **不可逆**(确认即冻结) |
| **2 生成材料** | Agent 本轮能看见什么:表结构、种子因子、算子白名单、硬约束 | 确定性列举 | 白名单外字段 fail-closed | 可回退 |
| **3 候选池** | 全部候选,**含被拒的** | LLM 生成 | 全部候选在场 | 可回退 |
| **4 校验** | 每个候选通过/拒绝 + 原因 | 规则层(确定性)→ 语义查重(LLM) | 两层都过才进 5,拒绝原因可查 | 可回退 |
| **5 训练+验证迭代** | 训练段明细全开;每改一轮消耗一次预算 | 确定性回测 + LLM 读结果提改法 | 预算余额 append-only 可查 | 可回退,**但预算不退** |
| **6 测试段拆封** | 一次性 | 确定性回测 | 拆封记录一次性写入 | **不可逆** |
| **7 结论** | 正 / 负 / 证据不足,同一入口 | 确定性裁决 | 附"共尝试 N 次" | 终态 |

**触发式卡片(不占流程段):**
- 回测完成 → 稳健性三卡(参数邻域平坦度 / 分年度一致性 / 中性化前后差异)
- 中性化完成 → 残差正交性卡(因子与行业哑变量、与市值的相关性 ≈ 0)
- 候选被拒 → 拒绝原因归因卡(未知字段 / 未来函数 / 近似重复 / 不可回测)

## 3. 最难的一段:验证之后怎么优化才不算作弊

核心矛盾:**验证结果出来后再去优化,相当于用测试集训练。** 但又确实需要一个迭代优化环。解法分三层:

1. **三段切分**(训练/验证/测试),不是两段。反复拿验证段做"继续/停止"决策,验证段一样会被榨干(ML 里的验证集过拟合)。
2. **尝试预算**(比三段更进一步):
   - 训练段:无限次,随便看随便调。
   - 验证段:每个假说 2 次预算(硬上限),全局也有总额。每次改动消耗一次,**余额显示在界面上**,用尽即该假说结案,换新假说。
   - 测试段:封存,拆封一次,只盖章,连"继续/停止"都不许参与。
3. **把余额和封条做成界面上的实体,不是文档里的纪律。** 界面有"本假说剩余尝试 1/2"计数器,和一道点下去有确认弹窗的封条。最终报告自动带"本结论前共尝试 N 次" —— 多重检验次数不藏起来,让读者自己折算(落地 #65)。

**副产品:** 预算逼出的正确做法是**看参数邻域平坦度**而非找最优点。那张图(横轴参数、纵轴 IC,平台 vs 尖峰)把"过拟合"变成肉眼可见的形状,比任何单个夏普数字都好讲。孤立尖峰(37 天最好)是过拟合自白;20/40/60 天都在 0.03~0.04 才可信。

## 4. Canvas:第一等公民是"对照卡",不是图表

要点:**不是图像的堆叠,要说清比对哪两个方面。** 所以 Canvas 的原语不是 `addChart()`,而是:

```
compare(处理组, 对照组, 在什么指标上, 控制了什么)
```

每张卡强制填满四个槽,填不满就画不出来。这恰好是本项目主实验的形状:处理组 = 结构化 + 文本,对照组 = 只有结构化,指标 = Rank IC 差值,控制 = 同一批股票、同一时间窗、同样中性化。

**AI 对画布的权限是白名单动作集**(`show` / `compare` / `annotate` / `split`),**绝不允许它产生任何数值**——所有数字来自确定性产物。结构对称,好讲:

> **白名单算子约束数值空间,白名单画布动作约束展示空间。** 两处用同一种手法保证可审计。

形态参考 Gemini / Claude Code 的 Canvas:一个 AI 有权限操作、但只能摆放确定性产物的画布。

## 5. "别的因子怎么去除的" — 可靠、简短、不喧宾夺主

默认只显示一行:**「行业/市值已剥离 · 残差相关性 0.003」**(绿色,一行字)。点开才展开三层:

1. **一句话机制**:把因子信号里的行业成分、市值成分投影出去,只留正交残差 —— 本质与傅里叶滤波同构,是一个带阻滤波器。
2. **凭什么可靠**:不靠声称,靠一个当场能跑的数字 —— 中性化后因子与行业哑变量、与市值的相关性应接近零,那个 0.003 就是它。
3. **边界(必须写)**:行业/市值这组基是每天估计的、彼此不正交、不能完美重构,所以是**削弱**不是**消除**。写上它反而更可信,且只占一行。

## 6. 拒绝多 Agent 架构(Factor GPT / Code GPT / Eval GPT)

参考过这套主流三 Agent 架构,**明确拒绝**,理由是硬红线不是复杂度:

- **Eval GPT 直接撞铁律 1** —— 评价必须是确定性代码。让 LLM 参与评价,整套装置公信力当场归零。
- **Code GPT 撞受限表达式引擎** —— 引擎前提是禁止任意代码执行、只允许白名单算子;引入代码生成等于拆掉安全边界。
- 剩下的 Factor GPT 就是现有 `generate.py`。

**答辩素材:** 多 Agent 增加的是能力,减少的是可审计性;而本项目的资产恰好是后者。全自动闭环已有(`loop.py`),但必须带迭代上限 —— 无上限的自动闭环就是自动化的 p-hacking。

## 7. 预设案例 = 主线,不是通用轻量案例

**"不一定通用,但预设案例要能跑通"** —— 这是本项目类似"模型训练"的那部分,跑出的产物再映射到论文。

- 预设案例**必须是主线**:业绩预告文本增量检验(ADR-0022,论文的可证伪断言本身)。若智能体真跑的是另一个轻量通用案例,主结论与"智能体"就成两条不相交的线,答辩一问即穿。
- 轻量通用案例价值只剩管道冒烟测试(先跑通管路),不是那个案例。
- **"不一定通用"的边界:预设的是入口,不是路径。** 可让的是**输入**(线索可以是我们写好的一句);不可让的是线索之后每一步(生成走真 LLM+真知识库,校验走真规则,回测走真代码,裁决走真阈值)。
- **判据:** 把预设线索换成一句没见过的话,管道应照样走完(结论可能很难看,但不能崩、不能走不到底)。走不完就是硬编码,是这条线上唯一不能妥协的红线。

## 8. 智能体是被建造的,不是被展示的("模型训练"类比)

真正的分野不是"生成智能体 vs 生成报告"(报告是产物,不是候选项),而是:

> **智能体是被建造的东西,还是被展示的东西。**

按"模型训练"类比,层次自动清楚:

| 类比 | 本项目 |
|---|---|
| 模型 | 闭环智能体在预设案例上的一次真实运行 |
| 训练产物 / checkpoint | run record(含全部候选、被拒原因、裁决) |
| TensorBoard | 面板(panel) |
| 论文 | 对那次运行的分析 |

**现状洞:** 项目叫"生成智能体",而 `results/factors/` 里只有 `llm_mock_*` 两个 mock 产物 —— 那个智能体从未真正生成过一个因子。这是首要待补的洞。

**因此录播式工作台是正解,不是造假:** 离线跑真实闭环 → 导出 run record → 面板回放。造假是当前状态(展示一个从未真跑过的智能体)。诚实性只缩到一句:**回放必须标成回放**(页面写明 run_id 与生成时间),不许包装成台上现算。见 ADR-0025。
