# MODEL_STRATEGY — 模型选择与调参策略

本项目的模型交付不是“让大模型直接预测收益”,而是“让模型提出可解释因子假设,再由确定性代码裁决”。因此模型调参的目标是提高候选因子的结构稳定性、字段合规性和经济解释质量,不是让模型自己计算 IC 或收益。

## 推荐交付形态

1. **默认可复现版本:** `mock` 后端。用于展示系统闭环、测试和答辩演示。
2. **API 版本:** OpenAI-compatible 后端。适合生成更丰富的候选因子和反馈建议。
3. **本地模型版本:** Qwen 等中文 instruct 模型通过 vLLM/Ollama 暴露 OpenAI-compatible 接口。适合说明可私有化部署。

## 为什么不急着微调

微调主要解决“输出风格和格式稳定性”,不解决真实数据、点时间对齐和回测正确性。对当前阶段而言,更高收益的改进通常是:

- 更清晰的字段字典;
- 更严格的 JSON schema 和解析重试;
- 更强的规则校验;
- 更好的回测报告反馈模板;
- 更多真实、人工审核过的候选因子样本。

## 调参顺序

1. **Prompt 调参:** 固定输出 JSON,明确可用函数、禁止未来字段,加入 2-4 个高质量示例。
2. **解码参数:** `temperature=0.1-0.3`,限制 `max_tokens`,避免长篇发散。
3. **规则层调参:** 字段存在性、表达式 AST、安全函数参数、未来收益字段、语义重复。
4. **闭环调参:** 根据通过率、IC 分布、换手率和基线相关性调整生成约束。
5. **LoRA 微调:** 只有在积累了高质量样本后再做,目标是稳定输出格式和因子风格。

## LLM 生成稳定性审计

生成阶段只负责提出候选假设,不读取真实收益路径或样本外结果。每个候选会记录生成参数和谱系信息:

- `generation.params`: 后端、模型、`temperature`、`max_tokens` 等可复现实验参数;
- `generation.proposal_rank`: 同一轮输出中的候选排序;
- `generation.generated_at_utc`: 生成时间戳;
- `source_seed_factors`、`synthesis_method`、`mechanism_change`、`expression_version`、`lineage`: 从种子因子到新候选的机制演化记录。

稳定性审计采用“确定性校验优先”的原则:先用 JSON 解析、字段白名单、表达式 AST 限制、未来字段禁用、时间窗口限制和基线相关性检查淘汰明显问题,再把通过规则的候选交给回测。LLM 可以解释为什么提出某个机制,但不能声称 IC、收益或显著性数值。

## 超参数与鲁棒性

回测默认参数不是由 LLM 搜出来的,而是作为确定性策略显式记录到报告中。当前策略包括:

- 交易成本网格: `0, 5, 10, 20, 30, 50, 100` bps;
- 分组数网格: `3, 5, 10` quantiles;
- 持有期/预测窗口网格: `fwd_ret_1, fwd_ret_5, fwd_ret_20`;
- 表达式复杂度上限: AST 节点数、AST 深度和最大时间窗口;
- 允许函数集合: 与因子引擎 `FACTOR_FUNCTIONS` 保持一致。

当前报告会输出相似性风险、训练/样本外落差、walk-forward 稳定性、早晚样本外分层、交易成本敏感性,并对尚未完整执行的维度显式标注 `not_tested_requires_*`,例如多股票池、行业内分层和完整持有期网格。这样可以防止把未做的鲁棒性检验包装成已完成结论。

## LoRA 是什么

LoRA 是 Low-Rank Adaptation,一种参数高效微调方法。它不是激活函数,也不是损失函数。直观理解是:冻结底座模型,只训练少量低秩适配矩阵,让模型更贴近特定任务的输出风格。

## 何时值得做 LoRA

满足以下条件时再考虑:

- 已有数百到数千条高质量 `instruction -> JSON response` 样本;
- prompt 已经调过,但 JSON 可解析率仍不稳定;
- 模型经常编造字段或生成不符合表达式语法的候选;
- 本地部署和答辩展示需要体现“轻量微调能力”。

## 最小 LoRA 数据格式

```json
{
  "instruction": "根据字段字典生成2个可解释A股因子,输出JSON数组。",
  "input": {
    "fields": ["eps", "close", "net_income", "total_equity"],
    "existing_factors": ["baseline_value_ep_bp"]
  },
  "output": [
    {
      "name": "quality_roe_rank",
      "expression": "rank(safe_div(net_income, total_equity))",
      "economic_rationale": "ROE更高可能代表资本使用效率更强。",
      "fields_used": ["net_income", "total_equity"]
    }
  ]
}
```

## 评估指标

微调后不要只看 loss,应看工程指标:

- JSON 可解析率;
- 字段命中率;
- 表达式可执行率;
- 未来字段违规率;
- 与已有因子重复率;
- 规则校验通过率;
- 通过校验后的样本外回测表现分布。

## 底线

回测里绝不调用 LLM。模型可以提案,可以解释,可以建议下一轮方向,但不能产生最终数值结论。
