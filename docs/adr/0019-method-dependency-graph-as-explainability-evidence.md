# 0019 — 方法依赖图作为"可解释性"的证据,而非装饰

日期:2026-07-21
状态:已接受

展示面新增的知识图谱,内容确定为**方法依赖图**:节点分层为 `原始字段 → 对齐与预处理 → 白名单算子 → 因子类别 → 评价指标`,边为**计算依赖**(`computes`)与**指标派生**(`derives`),防线(防 look-ahead / 样本外滚动 / 禁随机打乱 / 成本扣减)作为横切徽章挂在相关节点上。

点击任一节点高亮其**全部上游祖先直至原始字段**,并在面板中逐层列出该回溯链路。

## 理由

- 论文 §方法 的核心主张是:因子的可解释性"可以一路回溯到具体字段与有限算子,做到真正可审计"。**这张图就是该主张的可视化证据** —— 它不是装饰,是论点的现场演示。
- **一鱼三吃**,是平台侧唯一同时喂三个真交付的产物:站点交互图 / PPT 静态图(PowerPoint 原生吃 SVG)/ 论文配图(论文目前只有文字声称"可回溯",缺图)。
- **防抄风险低**:展示的是方法的*形状*,不是可复制的实现。炫耀价值高、被抄价值低。

## 考虑过的替代方案

- **概念先修图**(节点是术语,边是"要懂 B 先懂 A"):偏教学科普,对答辩不如方法依赖图锋利。其价值已被吸收为每个节点的 `one_liner`,不单独成图。
- **项目语汇图**(直接把 `CONTEXT.md` 的 15 个术语与 ADR 关系画出来):数据现成、做起来最快,但对外部观众价值低 —— 他们不关心 Defense Bundle 怎么定义。

## 边界

- **`factor` 层只放类别与基线因子名,不放任何具体因子表达式。** 图的结构本身已完整表达"可回溯",不需要暴露式子。**图证明的是"方法可审计",不是"因子是什么"。**
- 图数据**手写**于 `platform/content/graph.yaml`,不从代码自动抽取(自动抽取会把不想公开的字段名与真因子一并搬上去,破坏 ADR-0020 的 curation 闸门)。
- 防漂移测试**方向不对称**:只断言"图上的东西代码里有"(图 ⊆ 代码),**不断言反向**。反向断言等于强制所有代码实体上图,直接违反 curation。**图允许有意省略,不允许凭空捏造。**

## 后果

- 新增维护物 `graph.yaml`,需随算子/指标演进手工更新;`tests/test_graph_sync.py` 保证它不会捏造已不存在的代码符号。
- 布局在 `build.py`(Python)侧预计算并输出内联静态 SVG,**不引入 d3/cytoscape**:单文件站点内联图库体积不值,且力导向布局会把分层 DAG 甩成毛球,恰好毁掉要展示的层次。
- 布局必须确定性(平局按 id 字典序),否则每次构建产出不同 SVG,git diff 失效、导出物不可复现。
