# 0024 — 原展示站链接独立研究面板

日期:2026-07-23
状态:已接受(仅修订 ADR-0023 的“platform 不提供入口”条款)

项目所有者在研究面板完成后要求从原 `platform/` 站点提供入口。因此,原站成果区新增一条
公开内容卡片,链接到同域路径 `/panel/`。

这次修订只解决**可发现性**,不改变 ADR-0023 的架构边界:

- `panel/` 仍是独立 Vue 应用,有自己的依赖、源码、构建命令与 `dist/`;
- `platform/build.py` 不编译、不复制、不读取 panel 源码或数据;
- 两者不共享组件或运行时代码,只共用视觉语言;
- panel 的行情、事件、因子数值仍由 Python 在构建期导出为 JSON;
- 不为 panel 新增后端端点,不修改 `intake_service.py` 的职责;
- panel 构建产物独立部署到站点目录 `/panel/`,原站只持有一条普通链接。

## 为什么不是把 panel 并入 platform

原站是想法流与工单入口,其静态页面由 Python 生成;研究面板是交互密集的 Vue 应用。将两者合并会让
原站构建、工单服务和答辩演示相互牵连,也会削弱断网演示的确定性。普通链接足以提供统一入口,
没有必要建立代码或服务依赖。

## 后端演进

未来可以给 Agent 的自由输入增加受控后端,但必须另立 ADR,明确认证、限流、密钥保管、CORS、
可用性与成本边界。即使增加后端,预计算 JSON、预设 chip 和 P0/P1/P2 仍应保持静态可用,
数值计算仍不得交给 LLM 或运行期服务。后端只能作为可选增强层,不能成为答辩主路径。

## 后果

- 部署原站时继续发布 `platform/dist/index.html` 到站点根目录;
- 另将 `panel/dist/` 内容发布到 `/var/www/alpha.cihua.run/panel/`;
- `/panel/` 不可用时只影响研究面板入口,不影响原站与工单服务;
- ADR-0023 的其他约束继续有效,尤其是红线字段拦截、复权价格和
  `publ_date` / `usable_from` 分离。
