# 0001 — 项目为脊柱,平台是自动生成的副产物

日期:2026-07-20
状态:已接受

## 背景

暑期实训只剩 **5 天**,单人 + vibe-coding。个人目标有三:证明实力(开发)、练协作(workflow)、交友(向其他组分享中间节点)。原计划把"协作/分发平台"(参考 intelligrow-flow)做成与研究项目并列的第二产品。

## 决策

时间按约 **7:3** 分配:研究项目(alpha-factor-agent, M0→M6 压缩版)为脊柱;平台只做**最轻的壳**,其内容(节点记录/因子卡片)**从研究产物自动生成**,不为平台单独生产内容。

## 理由

- 答辩考核对象是研究项目;项目砸了,交友就没有可分享的硬货。
- 5 天内自写一个带状态流转的完整工单系统,会吃掉研究项目的可信底座(M0–M3)。
- "内容自动生成"让平台随研究进展免费更新,交友素材是研究的副产品而非额外负担。

## 被否决的替代方案

- **两条腿并重(5:5)**:大概率两头都是半成品。
- **平台先行一天,研究全程直播**:研究只剩四天,底座(防 look-ahead 单测、回测)质量不可接受。
