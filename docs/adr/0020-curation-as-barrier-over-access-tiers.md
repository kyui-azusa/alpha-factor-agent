# 0020 — curation 即壁垒:放弃分层权限体系

日期:2026-07-21
状态:已接受(取代设计过程中一度成型的四层 tier 方案)

展示面对"不全交底"的实现方式,确定为 **curation(内容策展)** 而非**访问控制**:

> `build.py` 只渲染写进内容文件的东西。**没写进去的,服务器上根本不存在。**

每个内容条目带 `public: true|false` 开关,**默认 `false`(fail-closed)**;论文条目另有 `detail_public` 开关控制是否提供 PDF 全文。前端的折叠/展开**只是叙事节奏,不承担访问控制**。

## 理由

- **任何 token 体系都比不过"它不在那儿"。** 对一个答辩即终结的项目,curation 是更强且成本为零的壁垒。
- 设计过程中曾成型一套四层体系(L1 公开 / L2 微信软门 / L2.5 口令 / L3 永不出)。它保护的是**未来的持续价值** —— 而**答辩即项目终点,没有未来价值可保护**,该体系的经济基础不成立。
- 微信软门被识破为伪命题:**能拿到网址的人,联系方式本就已有**,收集不到新信息;且填任意内容即可通过,拦不住任何人。
- ADR-0001 规定平台是副产品(Platform Byproduct)。距答辩仅 3–4 天且模型形态未定、论文仍是初稿,权限体系的工程量会直接挤占主线。

## 关键区分:红线安全 ≠ 防抄安全

站点的首要动机是**课程期同学分享面**,因此壁垒的真实含义是"别让同学在他们自己答辩前抄走方法"。这与 ADR-0003/0007 的红线是**两套判断**:

| | 管什么 | 论文全文的判定 |
|---|---|---|
| **红线**(ADR-0003/0007) | 别漏真 alpha / 真数据 | **安全** —— 全篇合成数据 + mock 因子,不含真结果 |
| **防抄**(本 ADR) | 别让方法被提前复制 | **不安全** —— 它就是完整方法论 |

故:**答辩前论文只公开摘要 + 架构图 + 章节目录("菜单"),不上传 PDF 全文。** 这同时遮住了 AI 初稿的粗糙 —— 现阶段放全文反而拉低可信度。

## 后果

- **不新增任何后端端点、不引入 token、不引入 tag/manifest/GitHub Contents API。** `intake_service.py` 除工单 milestone 字段外不动。
- 禁止任何"扫描目录自动收录""从代码自动抽取"的构建逻辑 —— 自动化会忠实地把不想公开的内容一并搬上去,是 curation 的天敌。
- **禁止把未公开内容渲染成"模糊占位块"**:那等于宣告"这里有东西",反而招惹。不存在的东西不留痕迹。
- 壁垒强度依赖人的判断而非系统约束:每次新增内容文件都要主动决定 `public` 的值。默认 `false` 是对这一弱点的兜底。
- 若项目将来延续(本 ADR 的前提失效),四层 tier 方案的完整评审记录保存在 `docs/PLATFORM_V2_SPEC.md` §5,可据此复活。
