# 0024 — 单支深色 Manim 正片,亮暗主题各用一张封面

日期:2026-07-23
状态:已接受

首屏方法讲解保留**一支深色 Manim 视频**。页面的深色与浅色主题各自使用一张专门渲染的
静态封面,主题变化时只更新原生 `<video>` 的 `poster` 属性。

不为两个主题分别导出正片。双份视频会让渲染、校对、体积与发布成本翻倍,还会引入两份内容
逐渐不一致的风险;用户切主题时若换 `src`,也会丢失当前时间、缓冲和播放状态。当前深色画布在
浅色页面中作为一块固定媒体画布是可接受的,主题适配的收益主要发生在尚未播放的封面状态。

## 连续性约束

- 主题切换只允许修改 `poster`,不得修改 `src`、调用 `load()` 或替换 `<video>` 节点。
- 已有的播放位置、暂停状态、音量和缓冲必须保留;播放中的主题切换不得中断影片。
- 初始 HTML 以深色封面作为无脚本回退,脚本再按页面实际主题同步封面。
- 两张封面来自同一个 Manim 静态场景,共享构图和文案,只替换背景、前景与弱化色。

## 重新评估条件

只有当真实用户测试证明深色正片嵌在浅色页面中存在持续的可读性问题,并且固定黑色媒体画布、
边框与封面适配仍不能解决时,才重新评估双主题正片。届时也必须先定义如何无损继承播放进度与
状态,不能把主题切换变成一次播放器重启。
