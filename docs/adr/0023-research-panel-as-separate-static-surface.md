# 0023 — 研究面板作为独立静态面,不并入现有展示站

日期:2026-07-23
状态:已接受

新增 `panel/` —— 一个独立的 Vue 应用,用于答辩现场演示"业绩预告事件 → 市场反应"的关联,
并支持自选股票/组合查看涨跌幅与 K 线。实现说明见 `docs/AGENT_PANEL_SPEC.md`。

**现有 `platform/`(想法流 + 工单入口,`alpha.cihua.run`)不做任何改动。** 项目所有者已确认
对其现状满意。两者是并列的两个面:platform 面向课程期同学做方法分享(ADR-0008 feed 为首屏),
panel 面向答辩现场做实证演示。共用视觉语言,不共用代码。

## 运行期不算数值,构建期算完

panel 是**纯静态**的:Python 在构建期跑完确定性回测并导出 JSON,浏览器只做读取、
组合加权(纯算术)与绘图。三条既有约束同时指向该形态:

| 约束 | 出处 |
|---|---|
| 回测里绝不调用 LLM | 铁律 1 |
| 不新增后端端点 | ADR-0020 后果条款 |
| 公开端点会被刷、消耗 token | 实际风险 |

**硬指标:`pnpm build` 后断网可完整演示。** 答辩现场不赌网络。这也顺带排除了"前端调后端实时
回测"的方案 —— 那既违反上表第二条,又把现场演示的成败绑在网络和服务上。

## LLM 由用户自带,项目不内置密钥

AgentConsole 的自由输入沿用 ADR-0013 的 OpenAI-compatible 口径,但**配置由使用者自己填**
(`baseUrl` + `apiKey` + `model`,存浏览器 localStorage)。项目不内置任何密钥、不代付 token,
因此上表第三条(公开端点被刷)自然消解,该功能得以从"有时间才做"提为正式条目。

两点约束:

- **LLM 只把自然语言翻译成视图参数**(输出结构化 JSON:看哪些股票、哪段区间、哪个因子),
  **绝不产生任何数值** —— 所有涨跌幅、IC、净值一律来自预计算 JSON。铁律 1 不受影响。
- **未配置时预设 chip 与全部图表照常可用。** 断网验收的口径是"预设 chip 必须可用",
  自由输入不可用是预期行为。

已知风险:panel 无后端,浏览器直连用户填的 `baseUrl`,**若对方未开 CORS 则无从代理**
(vLLM 需 `--allowed-origins`,Ollama 需 `OLLAMA_ORIGINS`)。故报错必须可操作,
不能只显示 "Network Error" —— 否则用户会误判为密钥填错,排查方向全错。

## 红线的落点与 ADR-0020 一致

panel 可展示行情、涨跌幅、K 线、预告披露日与已公开的结构化字段(类型/幅度区间)、
方法与算子白名单的名字、基线因子的类别名与回测图表。

**不得展示真实数据上的最终因子表达式与调优参数** —— 这正是 ADR-0020 修订后壁垒退守的位置。
该限制由导出脚本的字段白名单 + 对禁用字段的断言强制执行,**靠代码拦,不靠人记得**。

## 边界

- `usable_from`(披露日的下一交易日)与 `publ_date` 在数据契约里是两个字段,**不得混用**:
  K 线标记用 `publ_date`,任何"若当时买入"的计算一律从 `usable_from` 起算。混用即 look-ahead。
- 导出的行情必须是**复权后**价格。`data/raw/prices_raw.pkl` 不含 `adj_factor`,复权因子在
  `data/raw/adj_factors.csv` 且是事件记录,需 asof 展开后相乘(见 ADR-0021 与
  `sql/export_prices.sql` 注释)。
- `panel/public/data/` 除 `manifest.json` 外全部 gitignore:单只股票 7 年 OHLCV 约 100–150 KB,
  几百只即数十 MB,不进仓库(与 ADR-0003 的数据边界一致)。

## 后果

- 界面骨架取自 `chatzzy/ChatZZY`,但需明确:**那是未改动过的 `create-vue` 脚手架**,
  提供的是工具链配置而非可复用组件,示例组件需删除。
- 引入 `klinecharts` 一个图表依赖;净值曲线等普通图不再引第二个大库。
- 交付顺序为 骨架 → 数据导出 → ExplorerView → 断网验收,**时间不足时砍组合/因子/Agent 页,
  不砍断网验收**。距答辩仅 1.5–2 天且主线未完成,panel 是副产品(ADR-0001),不得挤占主线。
