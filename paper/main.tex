\begin{abstract}
\noindent
本文提出并实现一个\textbf{可解释 Alpha 因子生成智能体}，用以回答一个具体的研究问题:大语言模型（LLM）能否在\emph{不参与任何数值计算}的前提下，稳定地生成具备经济解释的 A 股 Alpha 因子。系统遵循“\textbf{LLM 提出假设、确定性代码负责计算与验证}”的分工范式——语言模型仅用于生成候选因子、辅助语义查重与阅读回测结果，所有关于收益、IC 与稳定性的数字一律由本地 Python 程序产生，从架构上杜绝“数值幻觉”与回测泄漏。
我们的贡献有三:(i) 给出行情、财务与股票池三表的\textbf{数据契约}，并把防前视（no-look-ahead）约束形式化为点时间（point-in-time, PIT）合并算子 \code{pit\_merge}，配以可证伪的单元测试;(ii) 设计一个仅允许白名单算子的\textbf{受限表达式引擎}，在支持因子可组合的同时禁止任意代码执行，使“可解释”与“可审计”落到实处;(iii) 构建一条\textbf{完全无 LLM 的确定性回测流水线}，产出 Rank IC、ICIR、分组收益、多空收益、换手率与成本后收益，并按日期滚动完成样本外划分。
在聚源 SQL Server 登录验证与 JYDB 元数据快照（2739 张表、81757 个字段）完成后，本文进一步导出行情、财务与股票池三张 raw 表，并在 2020--2021 年沪深 A 股 price-observed 股票池上重跑确定性流水线。初步样本外结果显示，部分基线因子在 2021 年具备正 Rank IC 与 ICIR，但两只 mock 生成候选因子未表现出稳健正向样本外收益，其中 \code{value\_profit\_blend} 还与既有基线高度相关。因此，本文的实证结论被限定为:可信底座已经能够在聚源真实数据上可复现运行，并能拒绝证据不足或新颖性不足的候选因子，而非宣称发现稳定 alpha。
\end{abstract}

\noindent\textbf{关键词：} Alpha 因子;可解释性;点时间对齐;样本外回测;大语言模型;确定性计算;防前视

\section{引言}
Alpha 因子研究的核心难点，从来不是让模型“说出一句听起来很聪明的话”，而是能否把一个候选假设转化为\emph{可计算、可验证、可复现}的研究对象。对量化研究而言，任何关于收益、信息系数（IC）或稳定性的结论，都必须来自可审计的数值流程，而不能是语言模型的口头裁定。近年来大语言模型被大量引入因子挖掘\cite{kou2025,alphaformer2026}，但一旦让模型直接“判断因子好坏”或“参与回测计算”，就会同时引入两类风险:\textbf{数值幻觉}（模型编造并不成立的统计量）与\textbf{回测泄漏}（在 $T$ 日的因子里混入了 $T$ 日之后才可得的信息）。这两类风险恰恰是既有实证文献反复强调的可复现性危机的放大器\cite{mclean2016,gkx2020}。

本项目从设计之初就以一条铁律划清边界:\textbf{回测里绝不调用 LLM}。语言模型负责提出因子想法、解释经济含义、阅读回测摘要;确定性代码负责字段可得性检查、点时间合并、表达式求值、样本外检验与报告生成（图~\ref{fig:arch}）。这样既保留了 LLM 的探索效率，又把所有数值判断交还给适合做计算、且可被逐行审计的模块。

\paragraph{本文贡献。}
\begin{enumerate}[leftmargin=2.2em,itemsep=2pt,topsep=2pt]
  \item \textbf{可证伪的防前视契约。} 我们把 no-look-ahead 约束形式化为点时间合并算子（\S\ref{sec:pit}，定义~\ref{def:pit}），并给出其防泄漏性质（命题~\ref{prop:noleak}）与对应的可证伪单元测试。
  \item \textbf{可审计的受限表达式引擎。} 因子被表示为携带经济解释与字段依赖的结构化对象，求值走白名单 AST，禁止任意代码执行（\S\ref{sec:engine}），使“可解释”不止于自然语言，而是可回溯到字段与算子。
  \item \textbf{完全确定性的回测流水线与端到端闭环。} 生成—校验—回测—反馈四步闭环中，唯有回测一步不碰 LLM（\S\ref{sec:agent}），并在聚源真实 raw 数据上完成初步样本外实证（\S\ref{sec:exp}）。
\end{enumerate}

\section{相关工作与定位}
机器学习驱动的资产定价已被系统研究\cite{gkx2020}，而学术因子被公开后收益衰减的现象\cite{mclean2016}提示我们:因子研究的价值高度依赖\emph{诚实的样本外评估}。近期一类工作尝试用 LLM 自动搜寻策略\cite{kou2025} 或以 Transformer 做 Alpha 因子的符号回归\cite{alphaformer2026}，展示了“语言/序列模型提出因子”的潜力。

与这些工作相比，本项目的定位不在“更强的搜索器”，而在\textbf{可信底座}:我们主张先把\emph{数据契约、防前视、样本外划分、确定性评价}这套地基做扎实、做成可测试的工程约束，再谈生成能力。换言之，LLM 的角色被刻意收敛为“假设提出者与结果阅读者”，其产出必须穿过确定性校验与回测才能被接受。这一保守设计以牺牲部分自动化程度为代价，换取了结论的\emph{可审计性}——我们认为这正是当前 LLM 因子研究最稀缺的性质。

\section{数据契约与点时间对齐}\label{sec:pit}
数据契约由三张表构成:\code{prices}（日频行情与复权因子 \code{adj\_factor}）、\code{fundamentals}（财务字段与公告日 \code{ann\_date}）、\code{universe}（历史成分股，用于避免幸存者偏差）。防前视的关键在于:财务数据必须按\emph{公告日} \code{ann\_date} 而非\emph{报告期} \code{report\_period} 对齐——一条报告期更早、但在 $T$ 日之后才公告的记录，在 $T$ 日是不可得的。

\begin{definition}[点时间可见集与因子面板]\label{def:pit}
对交易日 $T$ 与个股 $i$，记其\emph{点时间可见财务集}为
\[
  \mathcal{F}_{i}(T) \;=\; \bigl\{\, r \in \text{fundamentals}_i \;:\; \code{ann\_date}(r) \le T \,\bigr\},
\]
点时间合并算子 \code{pit\_merge} 取 $\mathcal{F}_i(T)$ 中\emph{公告日最新}的一条与 $T$ 日行情拼接，得到因子面板单元 $x_{i,T}=\phi\bigl(\text{prices}_{i,T},\,\arg\max_{r\in\mathcal{F}_i(T)}\code{ann\_date}(r)\bigr)$。因子在 $T$ 日的取值只能是 $\{x_{j,T}\}_{j\in\text{universe}(T)}$ 的函数。
\end{definition}

\begin{proposition}[防泄漏性]\label{prop:noleak}
若因子求值仅依赖 $\{x_{j,T}\}_{j}$，则对任意 $T$，其取值与所有 $\code{ann\_date}>T$ 的记录\emph{条件独立}，即不存在前视信息通道。
\end{proposition}
\noindent\emph{证明思路。} 由定义~\ref{def:pit}，$x_{i,T}$ 的构造在 $\mathcal{F}_i(T)$ 上取上确界，而 $\mathcal{F}_i(T)$ 按“$\code{ann\_date}\le T$”硬过滤，\allowbreak 故任何公告日晚于 $T$ 的记录都不在拼接来源中;又因子被限定为面板单元的函数，泄漏通道被完全切断。$\hfill\square$

该性质在实现中由 \code{pit\_merge} 强制执行（算法~\ref{alg:pit}），并配以\emph{可证伪}的单元测试（\code{test\_pit\_merge\_excludes\_future}）:构造一条公告日晚于 $T$ 的“未来”记录，断言它\textbf{绝不}出现在 $T$ 日面板中。测试失败即视为防线失守。

\begin{algorithm}[H]
\caption{\code{pit\_merge}:点时间合并（防前视核心）}
\label{alg:pit}
\begin{algorithmic}[1]
\Require 行情表 \code{prices}、财务表 \code{fundamentals}（含 \code{ann\_date}）
\Ensure 无前视的因子面板 \code{panel}
\ForAll{交易日 $T$，个股 $i \in \text{universe}(T)$}
  \State $\mathcal{F}_i(T) \gets \{\,r \in \text{fundamentals}_i : \code{ann\_date}(r) \le T\,\}$
  \If{$\mathcal{F}_i(T) = \varnothing$} \State 该单元置空并跳过 \EndIf
  \State $r^\star \gets \arg\max_{r \in \mathcal{F}_i(T)} \code{ann\_date}(r)$ \Comment{取最新已公告记录}
  \State $\code{panel}[i,T] \gets \text{join}\bigl(\text{prices}_{i,T},\, r^\star\bigr)$
\EndFor
\State \Return \code{panel}
\end{algorithmic}
\end{algorithm}

\section{方法}\label{sec:method}

\subsection{受限因子表达式引擎}\label{sec:engine}
因子以结构化对象 \code{FactorExpr} 表示，携带名称、表达式、\emph{经济解释}、字段依赖与元数据。表达式求值走\textbf{受限 AST 解析}:只允许基础算术与一组注册的白名单算子——\code{rank}/\code{cs\_rank}（截面排序）、\code{ts\_mean}/\code{ts\_std}（时序统计）、\code{delay}/\code{delta}（滞后与差分）、\code{safe\_div}（防零除）、\code{signed\_log}（保号对数）。白名单之外的一切节点（函数调用、属性访问、下标等）在解析期即被拒绝，从而\textbf{禁止任意代码执行}。这一设计的意义在于:因子的“可解释性”不再停留于自然语言的经济故事，而是可以一路回溯到\emph{具体字段}与\emph{有限算子}，做到真正可审计。

\subsection{基线因子库}
实现内置五类基线因子，覆盖经典风格:\emph{价值}（估值比率）、\emph{质量}（盈利能力）、\emph{动量}（短期趋势）、\emph{低波动}（收益波动）与\emph{流动性}（交易活跃度）。基线一方面构成最小可用研究集合，另一方面作为 Agent 语义查重与效果比较的参照系——任何 LLM 候选因子都要先与基线比对，避免“换皮重复”。

\subsection{回测与评价指标}\label{sec:metrics}
回测完全由本地代码实现。设 $f_{i,t}$ 为个股 $i$ 在 $t$ 日的因子值，$R_{i,t+1}$ 为其前向收益，$N_t$ 为当日截面样本数。核心指标定义如下。

\paragraph{Rank IC 与 ICIR。} 逐日计算因子与前向收益的秩相关，再对时间求稳定性:
\[
  \RankIC_t \;=\; \corr\!\bigl(\rank(f_{\cdot,t}),\,\rank(R_{\cdot,t+1})\bigr),
  \qquad
  \ICIR \;=\; \frac{\overline{\RankIC}}{\sigma(\RankIC)} .
\]

\paragraph{分组与多空收益。} 按截面分位数将股票分为 $Q$ 组，$G_k(t)$ 为第 $k$ 组;多空组合取最高减最低组:
\[
  r^{\mathrm{LS}}_{t+1} \;=\; \frac{1}{|G_Q(t)|}\!\!\sum_{i\in G_Q(t)}\!\! R_{i,t+1}
  \;-\; \frac{1}{|G_1(t)|}\!\!\sum_{i\in G_1(t)}\!\! R_{i,t+1}.
\]

\paragraph{换手率与成本后收益。} 以持仓权重 $w_{i,t}$ 的逐期变动近似交易摩擦，成本后收益按 \code{cost\_bps} 扣减交易成本:
\[
  \text{turnover}_t \;=\; \tfrac{1}{2}\sum_i \bigl|w_{i,t}-w_{i,t-1}\bigr|,
  \qquad
  r^{\mathrm{net}}_{t+1} \;=\; r^{\mathrm{LS}}_{t+1} - \text{turnover}_t \cdot \code{cost\_bps}.
\]
样本切分严格按日期滚动:\code{train\_end} 之后统一作为样本外区间，\textbf{禁止随机打乱}，以杜绝跨期信息混入。

\subsection{LLM 封装与 Agent 闭环}\label{sec:agent}
LLM 经统一客户端 \code{LLMClient.generate()} 封装，后端可切换（默认 OpenAI 兼容 API，测试用 mock），并对 prompt 做哈希缓存、限制 \code{max\_tokens} 以节省开销。闭环由三个 Agent 串起（算法~\ref{alg:loop}）:\emph{生成} Agent 产出结构化候选因子;\emph{校验} Agent 先过\textbf{确定性规则层}（字段存在、表达式可解析、是否遗漏 \code{delay} 等未来字段引用），再由 LLM 做\emph{语义查重};\emph{反馈} Agent 依据回测摘要建议改写方向。无论哪一步，最终数值一律以确定性回测为准。

模型调参的交付重心不是训练一个直接预测收益的黑箱，而是让“候选因子提案器”更稳定、更合规。当前优先调 prompt 模板、JSON 输出约束、\code{temperature}、\code{max\_tokens} 与规则校验;当积累足够多人工审核样本且格式仍不稳定时，才考虑 LoRA 等参数高效微调。LoRA 在本项目中的目标是学习因子表达风格与结构化输出习惯，而不是替代回测计算 IC 或收益。

\begin{algorithm}[H]
\caption{生成—校验—回测—反馈闭环（数值均由确定性代码产生）}
\label{alg:loop}
\begin{algorithmic}[1]
\Require 数据面板 \code{panel}、基线因子集 $\mathcal{B}$、迭代上限 $K$
\Ensure 通过校验并完成样本外回测的因子卡片集合 $\mathcal{C}$
\State $\mathcal{C} \gets \varnothing$
\For{$k = 1$ \textbf{to} $K$}
  \State $F \gets \textsc{GenerateAgent}()$ \Comment{LLM:产出结构化候选因子}
  \If{\textbf{not} \textsc{RuleCheck}$(F)$} \State \textbf{continue} \Comment{确定性规则层:字段/可解析/防前视}
  \EndIf
  \If{\textsc{SemanticDup}$(F, \mathcal{B}\cup\mathcal{C})$} \State \textbf{continue} \Comment{LLM:语义查重}
  \EndIf
  \State $m \gets \textsc{Backtest}(F, \code{panel})$ \Comment{\textbf{纯代码}:IC/ICIR/多空/换手}
  \State $\mathcal{C} \gets \mathcal{C} \cup \{(F, m)\}$;\quad $\mathcal{B} \gets \mathcal{B} \cup \{F\}$
  \State $\textsc{FeedbackAgent}(m)$ \Comment{LLM:读摘要，建议下一轮改写}
\EndFor
\State \Return $\mathcal{C}$
\end{algorithmic}
\end{algorithm}

\section{实验:聚源真实数据下的初步实证}\label{sec:exp}
\paragraph{实验目的与边界。} 本节使用从聚源 JYDB 导出的真实 raw 数据重跑确定性流水线，检验数据契约、防前视合并、表达式求值、样本外回测与报告落盘能否在真实 A 股数据上闭环。由于当前只覆盖 2020--2021 年、股票池为基于行情观测构造的沪深 A 股 price-observed universe，且尚未纳入严格停牌、涨跌停、ST、行业中性化与完整复权处理，本文将结果定位为\emph{初步实证与系统验收}，不把单次运行解释为稳健 alpha 发现。

\paragraph{真实数据接入状态。} 项目已完成聚源 SQL Server 本地接入、脱敏诊断与 JYDB 元数据快照流程，真实账号、密码、主机与库名仅保存在本地忽略文件或环境变量中，不进入代码仓库与论文正文。元数据快照覆盖 2739 张表与 81757 个字段;在此基础上，本文导出三张 raw 表:行情表 \code{prices.pkl} 共 1,904,369 行、4,261 只股票，日期范围为 2020-01-02 至 2021-12-31;财务表 \code{fundamentals.csv} 共 164,881 行、5,705 只股票，公告日范围为 2018-04-04 至 2021-12-31，报告期范围为 2018-03-31 至 2021-09-30，且 \code{(code, report\_period, ann\_date)} 重复键为 0;股票池表 \code{universe.csv} 共 1,904,369 行、4,261 只股票，与行情日期范围一致。经 \code{pit\_merge} 对齐后的面板共 1,904,369 行、18 列，其中 \code{mktcap} 非空 1,904,328 行。当前 \code{mktcap} 由 PIT 可见的 \code{shares\_outstanding} 与当日收盘价相乘得到，行情复权因子暂取 \code{adj\_factor=1.0}，相关限制在讨论节中单独列出。

\section{讨论、局限与后续工作}
\paragraph{价值定位。} 当前版本的价值在于把\textbf{可信底座}推进到真实聚源数据上，而非宣称已发现稳定 alpha。实验表明:程序链路正确、接口稳定、防前视与样本外约束有效;同时，候选因子若缺乏样本外表现或与基线过度重合，会被确定性评价明确拒绝。

\paragraph{有效性威胁（threats to validity）。} (i) \emph{外部有效性}:当前只覆盖 2021 年一个样本外年度，且股票池为 price-observed 沪深 A 股，并非官方指数成分或严格可交易股票池;(ii) \emph{构造有效性}:暂未加入 ST、停牌、涨跌停、冲击成本、行业/市值中性化等更严格交易约束，\code{adj\_factor} 目前取 1.0，\code{mktcap} 由 PIT 可见股本与收盘价近似;(iii) \emph{设计取舍}:LLM 仅“提想法/读结果”，因子优劣不由其裁定——这一保守设计牺牲了自动化程度，换取对数值幻觉与回测泄漏的最大抑制。

\paragraph{后续工作。} 下一阶段的三项主线:(1) 用官方成分股、ST/停牌/涨跌停过滤与完整复权字段替换当前简化口径;(2) 引入更严格的稳健性检验，包括多期滚动、分年度、行业与市值中性化后的稳定性;(3) 在缓存、\code{max\_tokens} 与审计日志约束下接入真实 LLM 后端，扩大候选生成空间，同时保持回测完全确定性。

\section{结论}
本文给出可解释 Alpha 因子生成智能体的初版实现与聚源真实数据下的初步实证。系统在\emph{数据层}以点时间对齐防止 look-ahead（并给出可证伪测试），在\emph{方法层}以受限表达式引擎与纯代码回测保证可审计，在\emph{交互层}以 LLM 封装与三 Agent 闭环提升探索效率——同时以铁律确保回测绝不触碰 LLM。最新实验已完成 JYDB 三张 raw 表导出、真实数据面板构建与 2021 年样本外回测:部分基线因子取得正向样本外表现，而 mock 候选因子未通过收益与新颖性检验。这一结果并不宣称发现稳定 alpha，却证明系统已经能够在真实数据上诚实地产生、检验并拒绝候选因子。我们相信，\emph{先地基、后生成}的路线，是让“LLM 生成因子”这件事从“看起来聪明”走向“可被相信”的必要一步。
